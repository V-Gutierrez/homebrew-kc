# kc — macOS Keychain CLI

> A human-friendly CLI for macOS Keychain. Because `security find-generic-password -s service -a account -w` is not human-friendly.

## Overview

**kc** replaces the macOS `security` command with an intuitive CLI for managing secrets stored in the native Keychain. Zero external dependencies. Your secrets never leave the Secure Enclave.

**Why:** Every dev on macOS has API keys in `.env` files, `.zshrc`, or `.bashrc` — plaintext, git-committable, screenshot-visible. The native Keychain is the right solution, but `security` command UX is hostile. **kc** fixes that.

## Stack Resolution

- **Language:** Go 1.22+
- **TUI:** Bubbletea + Lipgloss (Charm ecosystem)
- **Distribution:** Homebrew Tap → Homebrew Core (v2)
- **CI:** GitHub Actions (build + release + tap update)
- **Target:** macOS only (Keychain is macOS-native)

## Features (v1 MVP)

### Core CRUD
```bash
kc get API_KEY              # Read a secret (copied to clipboard + printed masked)
kc set API_KEY "sk-proj..." # Store/update a secret
kc del API_KEY              # Delete a secret
kc list                     # List all keys (values masked)
kc list --vault kafka       # List keys in specific vault
```

### Vaults (Service Groups)
```bash
kc vault list                    # List all vaults
kc vault create staging          # Create a new vault
kc vault switch staging          # Set active vault
kc set DB_PASS "xxx"             # Saved in active vault (staging)
kc get DB_PASS --vault prod      # Read from specific vault
```

### Import/Export
```bash
kc import .env                   # Import from .env file → Keychain
kc import .env --vault prod      # Import into specific vault
kc export --vault prod           # Export as .env to stdout (for CI piping)
kc export --vault prod -o .env   # Export to file
```

### Interactive TUI
```bash
kc                              # Launch interactive mode
kc -i                           # Same thing, explicit flag
```
- Fuzzy search across all vaults
- Values masked by default, reveal on Enter
- Copy to clipboard on select
- Add/edit/delete inline
- Vim-style navigation (j/k/g/G)

### Shell Integration
```bash
kc env                          # Print `export` statements for active vault
kc env --vault prod             # Print for specific vault
eval "$(kc env)"                # Load into current shell
source <(kc env --vault prod)   # Same, specific vault
```

## Data Model

```
Keychain (macOS native, AES-256 via Secure Enclave)
  └── Service = "kc:{vault_name}"
        └── Account = key_name
              └── Password = secret_value
```

- **Vault** maps to Keychain "service" field (prefixed `kc:` to avoid collisions)
- **Key** maps to Keychain "account" field  
- **Value** maps to Keychain "password" field
- Default vault: `kc:default`
- Migration: `kc migrate --from zshrc-secrets` imports existing entries

## Security Considerations

- **No network calls.** Ever. kc is offline-only.
- **No config files with secrets.** Vault list is inferred from Keychain.
- **Clipboard auto-clear.** After `kc get`, clipboard clears in 30s (configurable).
- **No shell history.** `kc set` with no value arg prompts interactively (hidden input).
- **Audit log.** Optional `~/.kc/audit.log` — who read what when (no values logged).
